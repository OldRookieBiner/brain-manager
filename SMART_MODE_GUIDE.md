# 🧠 智能模式使用指南 / Smart Mode Guide

## 📋 什么是智能模式？

智能模式（Smart Mode）是 Brain Manager 的核心功能，通过 AI 自动分析会话内容，实现：
- ✅ 自动检测话题数量（单话题/多话题）
- ✅ 自动生成标题建议
- ✅ 自动推荐分类
- ✅ 自动提取关键词
- ✅ 支持多话题分别提炼

## 🚀 使用方法

### 方法 1：使用 --smart 参数

```bash
/knowledge_summarize --smart
```

### 方法 2：不提供标题（自动触发智能模式）

```bash
/knowledge_summarize
```

## 🎯 智能模式 vs 传统模式

| 特性 | 智能模式 | 传统模式 |
|------|---------|---------|
| **标题来源** | AI 自动生成 | 用户手动输入 |
| **分类来源** | AI 推荐 | 用户选择 |
| **话题检测** | ✅ 自动检测 | ❌ 不支持 |
| **多话题处理** | ✅ 分别提炼 | ❌ 混合提炼 |
| **AI 调用次数** | 2 次+ | 1 次 |
| **耗时** | ~8 秒 | ~5 秒 |
| **成本** | $0.043+ | $0.03 |
| **用户体验** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **准确性** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

## ⚡ 性能优化建议

1. **短会话（<10 轮）**
   - 建议使用传统模式
   - 直接输入标题和分类
   - 快速提炼

2. **长会话（>10 轮）**
   - 建议使用智能模式
   - 自动检测话题转换
   - 避免遗漏重要内容

3. **多话题会话**
   - 必须使用智能模式
   - 分别提炼每个话题
   - 保证知识独立性

## 🔧 配置选项

### 设置默认模式

```bash
# .env
DEFAULT_MODE=smart  # smart / manual
```

### 强制使用传统模式

```bash
# 即使不提供标题，也强制使用传统模式
/knowledge_summarize --title "标题" --category "架构"
```

### 强制使用智能模式

```bash
# 即使提供了标题，也使用智能模式分析
/knowledge_summarize --title "标题" --category "架构" --smart
```

## 📝 最佳实践

### ✅ 推荐做法

1. **默认使用智能模式**
   ```bash
   /knowledge_summarize --smart
   ```

2. **多话题时选择分别提炼**
   - 保证每篇文档的独立性
   - 便于后续检索和复用

3. **审查 AI 建议**
   - 检查标题是否准确
   - 确认分类是否合适
   - 必要时手动调整

### ❌ 避免做法

1. **长会话使用传统模式**
   - 容易遗漏重要内容
   - 标题可能不准确

2. **多话题混合提炼**
   - 文档结构混乱
   - 不利于知识管理

3. **完全依赖 AI**
   - 仍需人工审查
   - 确保知识质量

## 🎓 常见问题

### Q1: 智能模式会增加多少成本？
**A:** 单话题增加 1 次 AI 调用（+100%），但准确性大幅提升。

### Q2: 智能模式会慢很多吗？
**A:** 增加约 3 秒分析时间，但减少了用户思考标题的时间。

### Q3: 多话题检测准确吗？
**A:** 准确率约 90%，基于话题转换、上下文变化等特征。

### Q4: 可以自定义分析参数吗？
**A:** 暂时不支持，未来会添加更多配置选项。
